# Support

## How to Get Help

### Questions about the playbook content

**"I don't understand how to apply X principle."**

Open a GitHub Discussion in the `Q&A` category. Include:
- Which document or section is unclear
- What you're trying to do
- What you've tried already

### Bugs or errors in the documentation

**"This link is broken." / "This section contradicts that one."**

Open a GitHub Issue with:
- File and line number where the issue appears
- What you expected
- What you found
- (Optional) Suggested fix

Label: `bug`

### Ideas and suggestions

**"There should be a section on X." / "This template is missing Y."**

Open a GitHub Discussion in the `Ideas` category before opening a PR.
New sections or governance concepts require discussion before implementation.

See [`CONTRIBUTING.md`](./CONTRIBUTING.md) for the full contribution process.

### Proposing an ADL (transversal architecture decision)

Open a GitHub Issue with:
- Title: `ADL Proposal: [topic]`
- Why this decision affects multiple projects
- Draft of the proposed decision
- Alternatives considered

Label: `adl-proposal`

See [`lifecycle/ADL.md`](./lifecycle/ADL.md) for the process.

### Security concerns

**Do not open a public issue.**

Email: cfdelrio@gmail.com

See [`SECURITY.md`](./SECURITY.md) for the full disclosure policy.

## What this repository does not support

- **Implementation help for specific projects** — AURORA, Engage, ProdeCaballito, and Storysell issues belong in their own repositories.
- **General software architecture consulting** — Use the Discussion board for focused playbook questions only.
- **Requests to bypass CONSTITUTION rules** — The Constitution exists for good reasons. If you think a rule is wrong, open a Discussion and make the case.
