# 00 — Manifiesto

> El por qué antes del cómo.

## Qué creemos

Construimos productos que tienen que durar, crecer y ser entendibles por gente que no estuvo cuando se escribieron. Eso exige disciplina, no heroísmo.

Creemos que:

- **La claridad es una feature.** Código que no se entiende es deuda, aunque funcione.
- **La arquitectura se decide, no se acumula.** Lo que no se diseña, se improvisa; lo improvisado se paga.
- **El dominio manda.** La tecnología es un medio; el problema del negocio es el fin.
- **La spec es un acto de pensamiento.** Escribir qué vamos a hacer nos obliga a entenderlo antes de romperlo.
- **La IA amplifica el criterio, no lo sustituye.** Un mal criterio asistido por IA produce malas decisiones más rápido.
- **La calidad es un proceso, no un momento.** No se "agrega calidad al final"; se construye en cada paso.

## Para quién es esto

Para mí, para mi yo futuro, para cualquier colaborador y para cualquier agente de IA que trabaje en estos proyectos. Todos deben poder leer este repo y operar con el mismo criterio.

## Qué NO es esto

- No es burocracia. Cada regla existe para evitar un dolor concreto.
- No es dogma. Las reglas cambian con ADRs cuando dejan de servir.
- No es un manual de un framework. Es doctrina, independiente de la moda tecnológica.

## El estándar

> Si no podés explicar la decisión, no la tomaste: la sufriste.

Cada decisión importante deja rastro. Cada feature deja una spec. Cada cambio de arquitectura deja un ADR. Así el conocimiento no vive en una cabeza, vive en el repo.
